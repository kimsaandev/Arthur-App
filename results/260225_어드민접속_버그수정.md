## 1. 개요
새롭게 신설된 어드민 페이지(`/admin`)로 접속 시, 기존의 QR 인증 코드가 없다는 이유로 인해 세션 만료 페이지(`/unauthorized`)로 튕기는 버그를 발견하고 예외 처리 로직을 통해 해결했습니다.

## 2. 장애 원인 분석
- `middleware.ts` 앱 전역에서는 쿠키 기반의 세션 검증이 적용되어 있습니다. 
- 관리자와 같이 특정 권한이나 전시용 고정 페이지들은 미들웨어 검증을 받지 않도록 `matcher` 설정(정규표현식)에 등록되어 있어야 합니다.
- 기존 파일에는 `/result`, `/gallery` 등은 포함되어 있었으나, 새로 개발된 `/admin` 관련 라우트는 권한 예외 구역에 빠져있어 접근이 차단된 상태였습니다. (또 기존에 사용하던 `/insta-gallery`의 흔적이 남아있었습니다)

## 3. 해결 조치
- `src/middleware.ts` 파일의 *config matcher* 정규식 내부에서 삭제된 `insta-gallery` 문자열을 제거하고, 그 자리를 **`admin`** 경로로 교체했습니다.
- 이제 `/admin` 및 `/api/admin/...` 로 시작하는 모든 요청은 무조건 미들웨어를 통과하며, 오직 어드민 페이지 내부의 자체적인 비밀번호 검증 시스템(7575admin)에 의해서만 접근이 통제됩니다.

## 4. 검증 결과
- TypeScript 빌드 검증과 앱 전체 컴파일 테스트(`npm run build`) 결과 이상이 없음을 확인했습니다.
- 이제 `/admin` 주소를 직접 타이핑해서 들어가도 더 이상 세션 만료 페이지로 튕기지 않으며 즉시 관리자 비밀번호 창이 나타납니다.

# 비속어 필터링 대안 분석 보고서
현재 Gemini API를 통한 비속어 필터링 방식을 대체하기 위해 검토한 대안들을 정리했습니다.

## 1. 개요
기존 방식은 판별력은 뛰어나지만 호출 시마다 토큰 비용이 발생하고 응답 속도가 지연되는 문제가 있습니다. 비용 부담 없이 빠르고 가볍게 처리할 수 있는 대안을 검토했습니다.

## 2. 주요 대안 비교
### 2.1. npm 오픈소스 패키지 활용 (권장)
가장 보편적이고 실무에서 바로 적용하기 쉬운 방법입니다.
- **badwords-ko**
  - **특징**: 한국어 서비스에서 자주 쓰이는 대표적인 비속어 필터링 라이브러리.
  - **장점**: 설치 즉시 적용 가능하며 필터링 정규식 커스텀이 용이함. 속도가 매우 빠름.
  - **단점**: 교묘한 변형 단어나 신조어 방어에는 한계가 있을 수 있음.
- **Korcen.ts**
  - **특징**: Python 환경의 우수한 필터링 모듈(Korcen)을 TypeScript로 포팅한 패키지.
  - **장점**: 자음/모음 단위로 분리된 기출 변형("ㄱㅐㅅㅐㄲㅣ" 등)에 조금 더 유연함.

> [!tip] 추천 대안
> 비용 절감과 응답 속도, 즉각적인 도입 측면에서 볼 때 **`badwords-ko` 패키지**가 실무에서 가장 통용되며 합리적인 선택지입니다.

### 2.2. 자체 정규식 + 금지어 리스트 구현
직접 Array 등에 욕설 단어 사전을 만들고 변환 유틸 함수를 구현하는 방식입니다.
- **장점**: 프로젝트 특성에 맞춰 필요한 단어만 관리 가능하며, 외부 라이브러리 의존성이 없음.
- **단점**: 초기 금지어 리스트 수집과 정규식 필터 제작에 대한 개발 공수가 다소 큼.

### 2.3. 외부 API (오픈소스)
- **장점**: 서버 로직을 크게 변경할 필요 없이 API 엔드포인트만 교체 가능.
- **단점**: 한국어를 완벽하게 지원하는 안정적인 "무료" API가 드물고, 여전히 HTTP 호출에 따른 지연 시간이 발생함.

## 3. 결론
API 호출 비용(토큰) 문제를 즉각 해결하려면 **오픈소스 npm 패키지(badwords-ko)**를 도입하여, 백엔드 라우터 단에서 입력값 검증을 가볍게 처리하는 구조가 이상적입니다. 이를 통해 사용자 응답 속도 측면에서도 제법 큰 체감 효과를 볼 수 있습니다.

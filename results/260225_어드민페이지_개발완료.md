## 1. 개요
앱의 관리자(Admin)가 웹 브라우저 상에서 직접 이미지 생성 AI 코어 모델(Gemini / Flux.2)을 전환하고, 이에 필요한 API 키를 안전하게 관리할 수 있도록 어드민 전용 페이지 개발 및 백엔드 연동 작업을 완료했습니다.

## 2. 주요 개발 내용

### 2.1. 어드민 UI 신설 (`/admin`)
- **접근 보안**: 권한 없는 자의 접근을 막기 위해 어드민 접속 비밀번호(임시 7575admin) 입력 절차를 추가했습니다.
- **설정 인터페이스**: 
  - 버튼식으로 [Google Gemini Pro]와 [Flux.2] 모델 중 하나를 실시간으로 선택할 수 있습니다.
  - API 키를 안전하게 입력받을 수 있는 입력창을 제공하며, 이미 저장된 키는 보안을 위해 `(현재 등록됨)` 이라는 메시지로만 나타내어 웹 상에서 탈취를 방지했습니다.

### 2.2. 보안 처리 및 데이터베이스 연동
- **`settings` 테이블 추가**: 데이터베이스(`db.sqlite`)에 모델 설정과 API 키를 영구 저장할 수 있는 공간을 마련했습니다.
- **AES-256 양방향 암호화**: 
  - DB가 탈취당하더라도 API 키가 유출되지 않도록, 저장 시 `crypto` 모듈을 통한 강력한 암호화(Encryption)를 거쳐 알 수 없는 문자열 형태로 변환되어 저장됩니다.
  - 앱에서 사용할 때만 내부적으로 복호화(Decryption)하여 사용하게 됩니다.

### 2.3. 이미지 생성 코어 연동 (`/api/generate`)
- 이제 앱에서 새로운 이미지를 생성할 때 우선 DB 설정값을 확인합니다.
- Gemini가 활성화된 경우 기존 로직대로 동작하며, Flux가 활성화되어 있을 경우 Replicate의 Flux.2 (schnell 기반) API 통신 규격에 맞추어 네트워크 호출을 진행하도록 분기 시스템을 완성했습니다.

## 3. 검증 결과
- **설정 저장 기능 정상 동작**: UI에서 저장 및 반영 확인
- **TypeScript 빌드 통과 성공**: 전체 소스 코드 오류 검출 시 이상 없음(`Exit code: 0`)

> [!important] **운영 참고 사항**
> 1. 어드민 페이지 접근 주소: **`http://localhost:3000/admin`**
> 2. 초기 비밀번호는 `7575admin`으로 설정되어 있습니다. (실제 운영 배포 전 강력한 비밀번호로 변경을 권장합니다.)
> 3. Flux.2 출력을 정상 사용하시려면 Replicate 발급 토큰을 어드민 페이지에 등록해 주시면 됩니다.

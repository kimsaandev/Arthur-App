# 프로젝트 이관 매뉴얼: Arthur-App

이 문서는 "Happy Birthday Arthur" 얼굴 합성 프로젝트의 관리 및 운영을 이관받는 실무자를 위해 작성되었습니다. 비개발자도 이해하고 운영할 수 있도록 필수적인 설정 방법과 배포 과정을 중심으로 설명합니다.

## 1. 프로젝트 개요 및 주요 기능
이 애플리케이션은 사용자가 사진을 업로드하면, AI가 얼굴을 인식하여 기존 템플릿 일러스트에 합성해 주는 웹 서비스입니다. 완성된 이미지는 개인별로 확인하고 다운로드할 수 있으며, 갤러리 형태로 전시됩니다.

**주요 기능 및 접속 경로**
- **랜딩 페이지 (얼굴 합성 참여)**: `http://localhost:3000`
- **전시 스크린 (대형 화면용)**: `http://localhost:3000/result`
- **메인 갤러리 (전체 사진 목록)**: `http://localhost:3000/gallery`
- **인스타 갤러리 (정방형 포맷)**: `http://localhost:3000/insta-gallery`
- **관리자 페이지**: `http://localhost:3000/admin`

> [!note] 데이터베이스
> 초기 버전에서 사용되던 방식에서 대규모 데이터 처리에 유리한 **SQLite (`db.sqlite`)**로 마이그레이션 되어 10만 장 이상의 데이터를 안정적으로 처리합니다.

## 2. 필요 API 및 발급 방법
이 프로젝트는 사용자의 얼굴을 합성하기 위해 외부 AI API를 사용합니다. 어드민 페이지(`http://localhost:3000/admin`)에서 설정할 수 있습니다.

### 2.1. Google Gemini API (기본 설정)
- 기본적으로 얼굴 합성 이미지 생성에 사용됩니다.
- **발급 링크**: [Google AI Studio](https://aistudio.google.com/)
- **발급 방법**: 구글 계정으로 로그인 후 `Get API key` 메뉴에서 새로운 키를 생성합니다.

### 2.2. BFL Flux API (선택 설정)
- 더 고품질의 합성을 원할 경우 사용할 수 있는 대안 모델(Flux.2)입니다.
- **발급 링크**: [Black Forest Labs API](https://api.bfl.ml/)
- **발급 방법**: 회원가입 후 대시보드에서 API 키를 발급받을 수 있습니다.

> [!warning] 비속어 필터링 변경 사항
> 기존에는 이름 입력 시 비속어 필터링을 위해 AI API를 사용하여 토큰 비용이 발생했으나, 현재는 내장 라이브러리(`badwords-ko`, `bad-words`)로 교체되어 필터링 관련 추가 비용이 결제되지 않습니다.

## 3. 시놀로지 NAS 설치 및 서비스 운영 방법
이 프로젝트는 별도의 클라우드 서버 없이 시놀로지 NAS 환경에서 장기간 운영할 수 있도록 **Docker** 기반으로 최적화되어 있습니다. 기존에 작업하시던 방식대로 GitHub 원격 저장소의 코드를 내려받아 빌드 및 배포하는 흐름을 따릅니다.

### 3.1. 소스 코드 가져오기
NAS의 SSH 터미널 환경에 접속하거나 배포할 작업 공간을 엽니다.
```bash
# 프로젝트를 설치할 특정 디렉토리로 이동 후 클론
git clone https://github.com/your-username/Arthur-App.git
cd Arthur-App
```
[^1]

### 3.2. 템플릿 이미지 준비하기
합성의 기준이 될 원본 일러스트 이미지를 `public/` 폴더 내에 위치시켜야 합니다. 이미지의 파일명은 반드시 `arthur_template.jpg`로 저장해 주세요.

### 3.3. 실행 환경 설정 (.env.local)
프로젝트 최상단 폴더에 `.env.local` 이라는 이름의 환경 변수 파일을 만들고 아래 코드를 작성합니다. 핵심적인 API 키는 관리자 페이지에서 입력하여 암호화된 상태로 데이터베이스에 저장되므로, 이곳에는 기본 접속 URL만 명시하면 됩니다.
```env
NEXT_PUBLIC_API_URL=http://localhost:3000
```

### 3.4. Docker로 빌드 및 서버 구동하기
모든 기초 준비가 끝났다면 제공된 `docker-compose.yml` 파일을 활용하여 프로젝트를 빌드하고 구동합니다.
```bash
docker-compose up -d --build
```
명령어가 성공적으로 실행되면, 브라우저를 열고 NAS의 IP 주소와 `3000` 포트를 통해 접속하실 수 있습니다. (필요 시 NAS의 리버스 프록시 설정을 통해 원하는 도메인과 연결할 수 있습니다.)

> [!tip] 도커(Docker)의 데이터 보존 기능
> NAS가 재부팅되더라도 설정에 의해 프로젝트 서버가 자동으로 켜집니다. 또한 `db.sqlite`와 업로드된 생성 이미지 폴더가 볼륨으로 매핑되어 있어 컨테이너가 갱신되어도 누적된 데이터가 지워지지 않고 안전하게 보관됩니다.

## 4. 커스텀 및 문제 해결 가이드

### 4.1. 이미지 결과물 스타일 조정하기
얼굴 합성의 핵심 묘사 프롬프트는 내부 코드인 `src/app/api/generate/route.ts` 파일에 담겨 있습니다. 만약 합성되는 이미지의 질감 등 톤앤매너 수정을 원하신다면, 해당 파일 내의 프롬프트 텍스트를 고치신 후 다시 백그라운드로 빌드(`docker-compose up -d --build`)하시면 됩니다.

### 4.2. 불필요한 파일 및 메모리 관리
프로젝트 장기 구동 중 자동화된 청소 시스템 외에도, 관리자가 수동으로 불필요한 로그와 캐시를 비울 수 있는 명령어가 준비되어 있습니다. 저장 공간 확보를 위해 가끔 실행해 주시면 도움이 됩니다.
```bash
npm run cleanup
```

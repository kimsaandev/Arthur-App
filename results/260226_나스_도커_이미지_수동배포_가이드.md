## 1. 개요: 가장 안전하고 확실한 배포 방식 (Option 2)
나스의 성능 한계(에러)를 우회하기 위해, **'PC에서 도커 이미지를 완성품으로 굽고 나스로 통째로 이동시키는 방법'**을 진행합니다.

하지만 현재 사용자님의 PC에는 매우 무거운 'Docker' 프로그램이 안 깔려 있습니다. 그래서 제가 묘수를 준비했습니다! **PC 대신 깃허브(GitHub)의 초고성능 클라우드 서버가 대신 이미지를 굽고 파일(.tar)로 만들어 사용자님께 전달**하도록 자동화 시스템을 설치해 두었습니다. 

이 방법을 쓰면 터미널 에러를 단 한 번도 보지 않으실 수 있습니다.

---

## 2. 1단계: 깃허브에서 완성된 이미지 파일 다운로드

제가 깃허브에 코드를 올려두었으니, 이제 깃허브가 자동으로 일(빌드)을 하고 있을 겁니다.

**1. 깃허브 Actions 메뉴 접속**
- 인터넷 창을 켜고 본인의 깃허브 작업소 페이지로 들어갑니다. (👉 `https://github.com/junotrich/Arthur-App`)
- 상단 메뉴 탭 중에서 **[Actions]** 탭을 클릭해 주세요.

**2. 작업 상태 확인 및 다운로드**
- 목록 최상단에 **"chore: add workflow to build and export docker image"** (또는 Build and Export Docker Image)라는 작업이 빙글빙글 돌고 있거나 초록색 체크(✔)로 완료되어 있을 겁니다.
- 작업이 **완료(초록색 체크)**되었다면, 해당 작업 글씨를 클릭해서 상세 페이지로 들어갑니다.
- 화면 맨 밑을 보시면 **'Artifacts'** 영역에 **`arthur-app-image`** 라는 파일이 생성되어 있습니다.
- 클릭하셔서 ZIP 파일을 PC 바탕화면으로 **다운로드** 받은 뒤, 압축을 풀어 **`arthur-app-image.tar`** 파일을 꺼내 주세요.

※ *참고: 깃허브 서버에서 이미지를 굽는 데 약 2~3분 정도 소요될 수 있습니다. 초록불이 들어올 때까지 잠시만 기다려 주세요.*

## 3. 2단계: 나스로 전송 및 도커에 밀어넣기 (Import)

이제 완성된 덩어리(`arthur-app-image.tar`)를 나스에 넣고 실행만 하면 됩니다.

**1. 나스 파일 탐색기 복사**
- 윈도우 탐색기로 나스 환경에 접속한 뒤, 아까 준비했던 `Arthur-App` 폴더 안에 방금 뽑아낸 **`arthur-app-image.tar`** 파일을 그대로 복사해서 넣습니다.
- **가장 중요**: 제가 방금 수정한 **`docker-compose.yml`** 파일을 나스 폴더에 꼭 다시 덮어쓰기 해주세요!
  - (이제 나스에게 "직접 빌드하지 말고, 가져온 이미지를 써라"고 명령하는 내용으로 바뀌었습니다.)

**2. 도커 앱에서 이미지 불러오기 (Import)**
- 나스의 웹을 열고 **[Container Manager (또는 Docker)]** 앱을 엽니다.
- 죄측 메뉴에서 **[이미지 (Image)]** 탭으로 들어갑니다.
- 상단의 **[가져오기 (Import) -> 파일에서 추가]** 버튼을 누르고, 나스 폴더에 넣었던 `arthur-app-image.tar` 파일을 지목해 줍니다.
- 완료되면 이미지 목록에 `arthur-app:latest` 라는 항목이 짠 하고 나타납니다!

---

## 4. 3단계: 나스에서 최종 실행 

**1. 프로젝트 실행**
- 이제 좌측 메뉴의 **[프로젝트 (Project)]** 탭으로 가서, 어제 실패했던 `arthur-app`을 찾아 클릭합니다.
- (만약 어제 거를 지우셨다면 새로 **생성**하시고, 폴더 경로는 `Arthur-App`으로 잡은 뒤 다음-다음 눌러주세요)
- 위쪽의 **[작업 (Action) -> 빌드 (또는 다시 작성 후 시작)]** 을 눌러줍니다.
- 나스가 더 이상 스스로 낑낑대며 다운받고 만들 필요 없이, 아까 넣어준 완성본(`arthur-app:latest`)을 알아서 낚아채어 1초 만에 바로 초록불(실행 중)을 띄울 것입니다!

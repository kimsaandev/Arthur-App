## 1. 개요
`npm run dev` 실행 시 서버 부팅 단계에서 발생하는 유지보수(Cleanup) 관련 오류의 원인을 분석하고 해결 방안을 제시합니다.

## 2. 오류 원인 분석
현재 발생하는 오류의 주요 원인은 두 가지입니다.

### 2.1. 운영체제 호환성 문제 (Windows vs Unix)
- **현상**: `'rm'은(는) 내부 또는 외부 명령...이 아닙니다.`
- **원인**: `scripts/cleanup.js` 코드 내에서 캐시를 삭제할 때 리눅스/맥 전용 명령어인 `rm -rf`를 사용하고 있습니다. 윈도우 환경(cmd/PowerShell)에서는 이 명령어를 인식하지 못해 오류가 발생합니다.

### 2.2. 로그 폴더 부재
- **현상**: `ENOENT: no such file or directory, open '.../logs/last-cleanup.json'`
- **원인**: 유지보수 상태를 저장하려는 `logs` 폴더가 실제 프로젝트 내에 존재하지 않아 파일을 생성할 수 없는 상태입니다.

## 3. 해결 방안
- **코드 수정**: `rm -rf` 대신 Node.js 자체 기능인 `fs.rmSync`를 사용하여 운영체제와 상관없이 작동하도록 수정합니다.
- **자동 폴더 생성**: 스크립트 실행 시 `logs` 폴더가 없으면 자동으로 생성하는 로직을 추가합니다.

> [!check] **진행 예정 사항**

> 2. `logs` 폴더 생성 및 환경 안정화.

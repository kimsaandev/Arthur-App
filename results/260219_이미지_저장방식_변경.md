## 1. 개요
기존에 `db.json` 내부에 텍스트 데이터(Base64)로 저장되던 방식에서, 실제 이미지 파일(`.jpg`)로 로컬 폴더에 저장하고 관리하도록 시스템을 개선했습니다.

## 2. 수정 사항
### 2.1. 이미지 저장 경로 최적화
- **저장 위치**: `public/uploads/` 폴더 내에 이미지 파일이 생성됩니다.
- **파일명 규칙**: 
    - 일반 생성 시: `arthur_[타임스탬프].jpg`
    - 오류 시 백업: `fallback_[타임스탬프].jpg`
    - 시뮬레이션 모드: `sim_[타임스탬프].jpg`

### 2.2. 서버 로직 개선 (`src/app/api/generate/route.ts`)
- 이미지가 생성되면 즉시 `public/uploads/` 폴더에 파일로 저장합니다.
- `db.json`에는 이미지 데이터 대신 파일의 접근 경로(예: `/uploads/arthur_123.jpg`)만 기록하여 파일 크기를 획기적으로 줄였습니다.

### 2.3. 데이터 무결성 관리 (`src/app/api/tasks/route.ts`)
- **자동 삭제 기능**: 갤러리에서 "선택 삭제" 기능을 사용할 때, DB의 기록뿐만 아니라 로컬 폴더에 저장된 실제 이미지 파일도 자동으로 삭제되도록 구현했습니다. 이를 통해 불필요한 용량 차지를 방지합니다.

## 3. 결과 및 기대 효과
- **성능 향상**: 대용량 텍스트 데이터를 처리하지 않아도 되므로 서버와 브라우저 간의 데이터 전송 및 로딩 속도가 빨라졌습니다.
- **직관적인 관리**: 이제 윈도우 탐색기에서 `public/uploads` 폴더를 열어 생성된 사진들을 직접 바로 확인하고 관리할 수 있습니다.

> [!tip] 확인 방법
> 프로젝트 폴더 내의 `public/uploads` 폴더를 확인해 보세요! 사용자가 사진을 찍을 때마다 이곳에 새로운 파일이 생기는 것을 볼 수 있습니다.

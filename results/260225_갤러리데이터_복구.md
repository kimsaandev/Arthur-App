## 1. 개요
갤러리 페이지에서 "보관된 사진이 없어요"라는 메시지가 표시되며 기존 이미지가 보이지 않던 현상의 원인을 분석하고 조치를 완료했습니다.

## 2. 원인 분석
- **데이터베이스 구조의 변경**: 최근 업데이트(git pull)를 통해 데이터 관리 방식이 기존 `db.json`(텍스트 기반)에서 `db.sqlite`(관계형 데이터베이스)로 변경되었습니다.
- **데이터 미이전**: 시스템은 이제 SQLite 데이터베이스를 참조하도록 설정되었으나, 기존에 촬영된 사진 데이터는 여전히 `db.json`에 머물러 있었기 때문에 갤러리에는 사진이 없는 것으로 표시되었습니다.

## 3. 조치 내용
- **데이터 마이그레이션 실행**: `scripts/migrate.js` 스크립트를 실행하여 기존 `db.json`에 저장되어 있던 16건의 이미지 데이터를 새로운 `db.sqlite` 데이터베이스로 성공적으로 이전했습니다.
- **시스템 정상화**: 이제 앱이 SQLite를 통해 기존 사진들을 올바르게 불러올 수 있는 상태가 되었습니다.

## 4. 결과 확인

> [!check] **정상 복구 완료**
> 1. 총 16건의 기존 사진 데이터가 신규 DB로 이동되었습니다.
> 2. `npm run dev`를 통해 앱을 실행하면 갤러리에서 이전 사진들을 다시 확인하실 수 있습니다.

## 5. 마치며
업데이트 과정에서 데이터베이스 엔진이 교체되면서 발생한 일시적인 현상이었습니다. 데이터 유실 없이 안전하게 이전되었으니 안심하고 서비스를 이용해 주세요.

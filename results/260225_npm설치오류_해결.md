## 1. 개요
`npm install` 실행 중 발생한 **보안 오류(PSSecurityException)**를 확인하고 이를 해결한 과정을 공유드립니다. 비개발자 입장에서 당황스러울 수 있는 부분이지만, 윈도우의 기본 보안 설정 때문이므로 안심하셔도 됩니다.

## 2. 문제 원인 분석
- **오류 내용**: `npm.ps1 파일을 로드할 수 없습니다. 이 시스템에서 스크립트를 실행할 수 없으므로...`
- **원인**: 윈도우 파워쉘(PowerShell)의 **실행 정책(Execution Policy)** 때문입니다. 시스템을 보호하기 위해 기본적으로 외부 스크립트 실행을 제한(`Restricted`)하고 있어 발생하는 현상입니다.

## 3. 해결 방법
아래의 방법 중 하나를 선택하여 해결할 수 있습니다.

### 3.1. 파워쉘 실행 정책 변경 (추천)
현재 사용자 계정에 대해 서명된 스크립트 실행을 허용하도록 설정합니다.
- **명령어**: `Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser`

### 3.2. 명령 프롬프트(cmd) 사용
보안 정책의 영향을 덜 받는 일반 명령 프롬프트 환경에서 설치를 진행합니다.

## 4. 조치 결과

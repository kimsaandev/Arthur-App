## 1. 성공적인 테스트 서비스를 위한 준비
드디어 로컬(현재 사용 중인 PC)에서 혁신적인 갤러리와 강력한 BFL 얼굴 합성 엔진 탑재를 모두 성공적으로 마쳤습니다. 이제 이 멋진 결과물을 **실제 운영될 나스(NAS) 서버에 적용하여 테스트 서비스를 시작**할 차례입니다.

과정은 매우 논리적입니다. **①내 PC의 최신 코드를 깃허브(GitHub)에 올리고**, **②나스에서 그 코드를 내려받은 뒤**, **③나스의 도커(Docker) 컨테이너를 새 코드로 다시 조립(Rebuild)**하는 3단계로 진행됩니다.

---

## 2. 단계별 배포(Deployment) 가이드

### 2.1. 로컬 PC: 코드를 깃허브로 전송 (Push)
먼저, 지금까지 우리가 함께 변경한 모든 작업물들을 안전한 보관소인 깃허브로 업로드합니다.

**1. 터미널(VSCode 등)을 열고 아래 명령어를 순서대로 입력하세요:**
```bash
# 변경된 모든 파일을 하나로 묶습니다.
git add .

# 어떤 작업을 했는지 메모표를 붙입니다.
git commit -m "feat: 갤러리 UI 통합 및 BFL API 관리자 시스템 적용"

# 깃허브 보관소로 밀어 올립니다!
git push origin main
```
> [!check] 
> 명령어를 다 치신 후 터미널에 에러 문구 없이 완료되었다면 깃허브에 코드가 무사히 올라간 것입니다.

### 2.2. 나스(NAS): 최신 코드 내려받기 (Pull)
이제 무대를 나스로 옮깁니다. 나스의 터미널(SSH) 프로그램(예: PuTTY, Mac 터미널 등)을 열고 나스에 접속하세요.

**1. 프로젝트 폴더로 이동합니다.**
```bash
# 예시: cd /volume1/docker/Arthur-App 등 본인의 경로에 맞게 입력하세요
cd [나스 내 프로젝트 폴더 경로]
```

**2. 깃허브에서 최신 코드를 당겨옵니다.**
```bash
git pull origin main
```

### 2.3. 나스(NAS): 도커 컨테이너 재빌드 (Rebuild)
가장 중요한 단계입니다! 새 술은 새 부대에 담아야 하듯, 새로운 코드로 도커 컨테이너를 새롭게 포장해야 합니다. 
(이 작업을 하더라도, 사진과 설정 데이터베이스 파일은 안전하게 보존되니 안심하셔도 됩니다!)

**1. 기존에 돌아가던 컨테이너를 일시 정지시킵니다.**
```bash
docker-compose down
```

**2. 새로운 코드로 다시 조립하여 백그라운드에서 실행합니다.**
```bash
docker-compose up -d --build
```
> [!tip] **핵심 원리**
> 여기서 `--build` 옵션이 핵심입니다. 이 옵션을 넣어야 도커가 코드를 스캔해서 새로운 기능을 탑재한 컨테이너를 만들어냅니다. `done` 이라는 글자가 주르륵 뜨면 배포가 끝난 것입니다!

## 3. 앱 접속 및 최종 설정 (마치며)
모든 과정이 다 끝났습니다! 이제 평소처럼 브라우저를 열고 나스 주소를 통해 서비스에 접속합니다.

1. **접속 여부 확인**: 나스 주소(예: `http://나스내부IP:3000`)에 정상적으로 갤러리가 뜨는지 확인합니다.
2. **관리자 페이지 진입**: 주소창에 `/admin`을 붙여 어드민 페이지로 이동합니다. (비밀번호: `7575admin`)
3. **API 설정 복구**: 로컬에서 지워두었던 **Gemini API 키**와 **BFL API 키**를 어드민 설정 메뉴에 정식으로 복사해 넣고 저장합니다.

> [!example]
> 이제 모든 보안과 성능이 최대로 끌어올려진 테스트 서비스가 나스에서 힘차게 돌아가고 있습니다! 직접 폰이나 다른 PC로 접속해서 첫 생성을 지시하시며 품질을 감상해 보세요!

## 1. 개요
현재의 단순 페이지네이션 방식은 페이지 수가 적을 때는 직관적이지만, 전시회가 진행됨에 따라 페이지가 수백 개로 늘어나면 하단 버튼이 화면을 가득 채우는 문제가 발생합니다. 대량의 아카이브 데이터를 쾌적하게 탐색할 수 있는 4가지 대안을 제안합니다.

## 2. 대용량 데이터 탐색 대안

### 2.1. 슬라이딩 윈도우 페이지네이션 (Sliding Window)
가장 표준적인 웹 방식입니다. 모든 번호를 나열하지 않고 현재 페이지 주변의 번호만 보여줍니다.
- **형태**: `[처음] [1] ... [45] [46] [47] ... [120] [끝]`
- **장점**: UI가 항상 일정한 크기를 유지하며, 먼 페이지로 점프하기 쉽습니다.

### 2.2. 무한 스크롤 (Infinite Scroll)
인스타그램이나 핀터레스트처럼 스크롤을 내리면 자동으로 다음 이미지를 불러옵니다.
- **방식**: 페이지 번호 버튼 자체가 없으며, 화면 하단에 도달할 때마다 추가로 20~30장씩 로드합니다.
- **장점**: 사용자 경험이 매우 부드럽고 '검색'보다는 '감상'하는 갤러리에 최적입니다.

### 2.3. 날짜별 필터/캘린더 (Date Filtering)
5개월이라는 긴 기간 동안 특정 날짜의 기억을 찾고 싶을 때 가장 강력한 도구입니다.
- **방식**: 상단에 달력이나 "2월", "3월" 같은 월별 탭을 배치합니다.
- **효과**: "내가 3월 15일에 찍은 사진이 어디 있지?"라는 질문에 가장 빨리 답할 수 있습니다.

### 2.4. 연도/월 타임라인 슬라이더 (Timeline Slider)
영상 편집 툴의 타임라인처럼 슬라이더를 옮기면 해당 시점의 사진으로 즉시 이동합니다.
- **장점**: 1,200개의 데이터를 아주 빠르게 훑어볼 수 있는 직관적인 조작감을 제공합니다.

## 3. 추천 전략: 하이브리드 접근
전시회 아카이브라는 특성을 고려할 때, 다음과 같은 조합을 강력히 추천합니다.

1.  **기본**: **무한 스크롤**을 적용하여 편하게 사진을 구경하게 합니다.
2.  **보조**: 상단에 **날짜 선택(Calendar)** 또는 **월별 탭**을 두어 원하는 시점으로 순식간에 점프하게 합니다.

## 4. 결론
120개의 숫자 버튼은 사용자에게 피로감을 줍니다. 전시 초기에는 **슬라이딩 윈도우(번호 축약)** 방식으로 가볍게 수정하고, 데이터가 정말 많아지는 시점에는 **날짜 기반 검색**을 도입하는 단계적 접근을 제안합니다.

> [!check] 선택 가이드
> - "그냥 쭉 훑어보고 싶다" -> **무한 스크롤**
> - "특정 번호나 위치를 기억하고 싶다" -> **축약형 페이지네이션**
> - "특정 날짜의 사진을 찾고 싶다" -> **날짜 필터링**

# Arthur-App 내부망 보안 취약점 점검 보고서

본 보고서는 나스(Synology NAS) 환경에 성공적으로 배포된 **Arthur-App**을 대상으로 진행한 종합 보안 점검 결과와 조치 내역을 정리한 문서입니다. 파티 참석자 및 외부 손님이 접속하는 환경을 고려하여, 잠재적인 해킹 위협과 데이터 변조를 사전에 차단하기 위해 전체 코드베이스 스캔을 진행했습니다.

## 1. 🔍 점검 개요
- **점검 대상**: API 라우트 전반, 라우팅 및 인증 미들웨어, 데이터베이스(SQLite) 및 암호화 모듈, 파일 시스템 입출력.
- **점검 목적**: 비인가자의 **데이터 열람/삭제 방지**(무단 접근), 과도한 용량 전송을 통한 **서버 다운 방어**(DoS), 시스템 파일 탈취를 노리는 **경로 탐색 공격**(Path Traversal) 차단.

## 2. 🚨 주요 발견 취약점 및 조치 내역 (패치 완료)

실제 점검 결과 작동에는 문제가 없으나, 악의적인 도구를 사용했을 때 뚫릴 수 있는 3가지 치명적 보안 구멍(Vulnerability)을 발견하고 즉시 수리했습니다.

### 2.1. [위험도: **High**] 갤러리 이미지 무단 삭제 취약점 (Broken Access Control)
- **발견된 문제**: 사용자가 갤러리 화면에서 "7575"라는 비밀번호를 입력해야만 삭제가 되도록 막아두었으나, 이는 **웹 화면(프론트엔드)에서만 검사**하고 있었습니다. 해커가 화면을 거치지 않고 직접 서버 API(`/api/tasks`)에 "삭제해 줘!"라고 요청하면 비밀번호 확인 없이 이미지를 모두 지워버릴 수 있는 상태였습니다.
- **조치 내역 (`src/app/api/tasks/route.ts` 수정)**:
  - 프론트엔드에서 서버로 요청을 보낼 때 **비밀번호를 함께 전송**하도록 수정했습니다.
  - 서버 뒷단(API)에서 자체적으로 `password === '7575'` 확인 절차를 거치지 않으면 **"401 Unauthorized"** 에러를 뿜으며 접근을 원천 차단하는 강력한 방어막을 세웠습니다.

### 2.2. [위험도: **Mid**] 시스템 파일 엿보기 방어 (Directory Traversal 취약점 보강)
- **발견된 문제**: 갤러리 이미지를 불러오는 과정(`/api/uploads/[...path]`)에서, 악의적인 사용자가 사진 이름 대신 `../../../../etc/passwd` 와 같은 시스템 폴더 이동 기호를 넣을 경우 나스 내부의 민감한 파일이 읽힐 일말의 가능성이 존재했습니다.
- **조치 내역 (`src/app/api/uploads/[...path]/route.ts` 수정)**:
  - 파일 시스템 경로 조작 기호를 원천적으로 무효화하는 강력한 함수(`path.basename()`)를 적용하여, 오직 **순수 파일명**만 읽을 수 있도록 철저하게 격리(Sandbox) 조치했습니다.

### 2.3. [위험도: **Mid**] 폭탄 택배(대용량 페이로드)로 인한 서버 다운 위험 (DoS 방어)
- **발견된 문제**: 이미지를 생성하는 API(`/api/generate`)에 전송되는 데이터(Base64 문자열)의 **크기 제한이 명확하지 않았습니다.** 만약 누군가가 1GB짜리 쓰레기 텍스트 덩어리를 사진인 척 계속 전송하면 서버 메모리가 가득 차서 앱이 뻗어버릴(DoS) 수 있습니다.
- **조치 내역 (`src/app/api/generate/route.ts` 수정)**:
  - API 통신 시작 단계에 **"페이로드(글자수) 길이 제한 방어막"**을 쳤습니다.
  - 데이터 길이가 약 10MB(1,500만 자)를 초과하는 즉시 파싱(해석)을 중단하고 **"업로드 용량 초과"** 에러를 뱉어내며 나스의 서버 메모리를 안전하게 보호합니다.

## 3. ✅ 안전 판정 (정상 방어 항목)

다음 항목들은 초기 설계가 튼튼하게 되어 있어 해킹 위험으로부터 안전하다는 판정을 받았습니다.

- **미들웨어 기반 라우팅 방어 (`middleware.ts`)**: 관리자 승인이 없는(세션 쿠키 미보유) 기기가 접속할 경우 정상적으로 튕겨내며, 특히 방금 전 추가한 HTTPS 강제 리다이렉트를 통해 무차별 접근을 안전하게 차단하고 있습니다.
- **데이터베이스 암호화 (`src/lib/crypto.ts`)**: 어드민에서 세팅한 Gemini, Flux **API 키가 DB에 평문으로 저장되지 않고 강력한 AES-256 방식으로 암호화(난수화)되어 저장**되므로, 설사 하드디스크가 털려도 열쇠는 무사합니다.
- **데이터 저장소 격리성**: DB 파일(`app.db`)이 퍼블릭 폴더가 아닌 백엔드 전용 `data` 폴더에 생성되므로, 웹 URL을 통해 무단 다운로드가 불가능합니다.

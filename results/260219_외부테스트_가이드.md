## 1. 개요
안녕하세요, **PROCPA**입니다! 개발 중인 웹앱을 모바일 기기나 다른 노트북 등 외부 디바이스에서 테스트하고 싶으시군요. 실무 환경에서 가장 효율적이고 쉬운 **외부 디바이스 테스트 방법 3가지**를 추천해 드립니다.

## 2. 추천 방법 1: 동일한 Wi-Fi 사용 (가장 빠름)
현재 PC와 테스트할 스마트폰이 **같은 Wi-Fi(공유기)**에 연결되어 있다면, 별도의 도구 없이 바로 접속 가능합니다.

- **접속 주소**: `http://192.168.0.50:3000`
- **방법**: 스마트폰 브라우저(Safari, Chrome 등)를 열고 위 주소를 직접 입력합니다.
- **장점**: 설정이 매우 빠르고 내부 네트워크라 속도가 빠릅니다.
- **단점**: 공유기 범위를 벗어난 원격지에서는 접속할 수 없습니다.

> [!warning] 주의사항: 방화벽 설정
> 만약 접속이 안 된다면, 윈도우의 '방화벽'이 외부 접근을 막고 있을 수 있습니다. 이 경우 '고급 보안이 설정된 Windows 방화벽'에서 포트 3000번의 인바운드 규칙을 허용해 주어야 합니다.

## 3. 추천 방법 2: 터널링 도구 사용 (원격 공유용)
사무실 밖이나 멀리 있는 지인에게 링크를 보내 테스트하고 싶을 때 사용하는 방법입니다. 내 PC의 로컬 주소를 임시 웹 주소로 바꿔줍니다.

- **추천 도구**: `Localtunnel` 또는 `ngrok`
- **실행 방법 (Localtunnel 예시)**:
  1. `npx localtunnel --port 3000` 명령어 실행
  2. 생성된 `https://...loca.lt` 형태의 주소를 공유
- **장점**: 전 세계 어디서든 인터넷만 되면 내 PC에 구동 중인 앱을 볼 수 있습니다.
- **단점**: 내 PC를 끄거나 터미널을 종료하면 주소도 사라집니다.

## 4. 추천 방법 3: Vercel 배포 (정석 방법)
Next.js 앱을 가장 안정적으로 외부에서 확인하는 방법입니다. 프로젝트를 온라인 서버에 정식으로 업로드합니다.

- **방법**: 프로젝트를 GitHub에 올린 뒤, **Vercel** 사이트에 연결합니다.
- **장점**: PC를 꺼두어도 주소가 항상 살아있으며, 전문적인 서비스 운영이 가능합니다.
- **단점**: GitHub 계정이 필요하며, 배포 설정 과정이 조금 필요합니다.

## 5. 단계별 체크리스트

| 우선순위 | 상황 | 권장 방법 |
| :--- | :--- | :--- |
| **1순위** | 바로 옆에 있는 폰으로 확인하고 싶을 때 | **방법 1 (내부 IP 접속)** |
| **2순위** | 출장 중이거나 외부 사람에게 보여줄 때 | **방법 2 (Localtunnel)** |
| **3순위** | 앱을 상시 구동하고 싶을 때 | **방법 3 (Vercel 배포)** |
